    üé® Key Design Principles¶

    1. Autonomous Activation¶

    Skills activate automatically based on user questions - no explicit commands needed.

    ‚ùå Bad: "Activate parallel-development-expert skill"
‚úÖ Good: "How can I work faster?"

    2. Educational, Not Just Functional¶

    Skills explain the "why", not just the "what".

    ‚ùå Bad: "Executing parallel workflow..."
‚úÖ Good: "This runs 3 tasks concurrently, saving 60% time because..."

    3. Quantified Impact¶

    All recommendations include concrete metrics.

    ‚ùå Bad: "This is faster"
‚úÖ Good: "This saves 2.3 hours (23% improvement)"

    4. Safety First¶

    No destructive operations without explanation and confirmation.

    ‚úÖ "This will delete X. Proceed? (Type 'yes')"
‚ùå *silently deletes*

    5. Natural Conversation¶

    Skills feel like talking to an expert, not reading documentation.

    ‚úÖ "Let me analyze your project... Found 3 parallelizable tasks!"
‚ùå "Analysis complete. Results: 3 tasks detected."
